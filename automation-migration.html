🚀 自動化測試轉型：Selenium 遷移至 Playwright

C# .NET 8 Playwright NUnit

專案背景 (Situation)

為了解決 Selenium 框架在執行效率與穩定性上的限制，評估並執行了框架遷移專案：

  • 效能瓶頸： 缺乏原生並行支援，測試套件運行時長過大。
  • 穩定性挑戰： 動態加載導致大量的 Flaky Tests，維護成本極高。

執行方向 (Action)

  • 技術 POC 評估： 對比自動等待機制與執行速度，驗證 Playwright 優勢。
  • 架構重新封裝： 採用 POM (Page Object Model) 重構，並統一跨瀏覽器驅動初始化。
  • 環境配置隔離： 實作動態環境配置管理，支援 Dev/Staging 無縫切換。

測試框架分層 (Architecture Layers)

1. Test Cases Layer (NUnit)

負責測試邏輯與斷言，僅呼叫 Page Objects，不涉及 UI 定位。

⬇️
2. Page Object Model (POM) Layer

封裝頁面元素定位 (Locators) 與 UI 交互行為。

⬇️
3. Core Utilities & Driver Engine

處理 Playwright Browser 實例、API 客戶端與資料庫連線。
