🚀 企業內部查詢平台重構 (Web Form to .NET Core)

專案概述： 既有內部查詢平台因使用 Web Form 舊架構，面臨 ViewState 負載過重導致載入緩慢、程式碼高度耦合難以測試等問題。本專案將核心邏輯遷移至 .NET Core 架構，提升系統擴充性與執行效能。

🛠️ 架構轉型對照 (Architecture Evolution)

重構前 (Legacy Web Form)
  • 高度依賴 ViewState，造成頁面 Size 過大。
  • 商業邏輯散落在 Code-behind (Page_Load)。
  • 手動處理資料庫連線與查詢，難以單元測試。
重構後 (.NET Core / MVC)
  • 改用 JSON 資料交換，大幅減輕伺服器負荷。
  • 導入 Service Layer 與 Repository Pattern。
  • 全面支援 Dependency Injection (DI) 提升靈活性。

🌟 關鍵技術貢獻

  • 非同步化優化： 全面導入 async/await 進行非同步查詢，解決原本同步執行時容易造成的 Thread Pool 阻塞問題，大幅提升高併發下的系統穩定性。
  • 中介軟體實作： 建立自定義 Middleware 統一處理例外錯誤 (Global Exception Handling) 與 操作記錄，降低程式碼重複率。
  • 效能提升： 經由架構翻新與資料檢索邏輯優化，複雜報表之查詢回應速度平均提升達 40% 以上。
