🔄 SAP 與 BPM 異質系統對接開發

專案目標： 實現 SAP 表單送簽流程自動化，將 ERP 端的單據透過中繼層同步至 BPM 簽核系統。

系統架構流圖 (System Architecture)

graph LR A[SAP 送簽] --> B{中繼 Table} B --> C[Control File GUID] B --> D[BPM 流程參數] B --> E[表單頭/單身] C & D & E --> F[批次發起 BPM 程式] F --> G[BPM 表單簽核] G --> H[結案 InBound 回傳] H --> I[更新 SAP 狀態]

🛠️ 技術實作與卡控點

  • 中繼層機制： 設計 SQL Server 中繼資料表，透過 Control File 與 GUID 確保每一筆採購單的唯一性與資料完整性。
  • Outbound/Inbound 閉環： 除了將 SAP 資料拋出，更設計了資料回補機制，當 BPM 簽核結束後，自動透過 Inbound 批次程式將狀態反寫回 SAP。
  • 異常通知機制： 針對公司、工號、部門等關鍵欄位設計轉換邏輯，若對接過程發生異常，系統會自動發出通知，確保流程不中斷。
